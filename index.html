1) Explain the Update Operation in Spring Data JPA.

  • Update Operation in Spring Data JPA:
    • The Update Operation in Spring Data JPA modifies an existing record in the database.
    • There are two ways to perform update operations:
      • Using save() Method
      • Using @Modifying and @Query

i) Using save() Method:

  • The save() method is used to update an existing record in the database.
  • When you call the save() method, JPA automatically performs an update instead of an insert.
  • In JpaRepository, the save() method is used for both inserting and updating records.

Example:

A.save(a);

ii) Using @Modifying and @Query:

  • The @Query annotation is used to update a specific record instead of the whole record.
  • The @Modifying annotation is used to update the records instead of reading them.
  • The @Transactional annotation is used to run the queries safely inside a transaction.
  • We use JPQL (Java Persistence Query Language) to execute these queries.

Example:

@Modifying
@Transactional
@Query("statement")

3) Explain SOA - Service Oriented Architecture in Web Services.

  • SOA stands for the Service-Oriented Architecture.
  • SOA is a way to build the software by using small parts called as services.
  • Each service does one job and can be used again in other programs.
  • There are mainly two major roles in SOA:
    • Service Provider
    • Service Consumer

a) Service Provider:

  • Creates and publishes services.

b) Service Consumer:

  • Finds and uses services.

Components of SOA:

  • Web Services:
    • Web Services is a standard way to implement SOA.
    • They allow applications to interact with the web by using XML-based messaging protocols.
  • Key Technologies:
    • SOAP
    • WSDL
    • UDDI
    • RESTful services

4) Explain Types of Web Services, RESTful web services and their creation.

  • Web Services:
    • Web Services is a Standard way to implement SOA.
    • They allow Applications to interact with the web by using XML-based messaging protocols.
  • There are mainly two types of web services:
    • a) SOAP-based Web Services
    • b) RESTful Web Services

a) SOAP-based Web Services:

  • SOAP stands for Simple Object Access Protocol.
  • SOAP is a protocol to send data between client and server by using XML.
  • SOAP is complex, slow and has high overhead.

b) RESTful Web Services:

  • It is not a protocol, and REST is a way to build web services.
  • RESTful Web Services are simple, fast and have less overhead.
  • It uses HTTP Methods like:
    • GET → Retrieves all users.
    • POST → Creates a new user.
    • PUT → Updates a user.
    • DELETE → Deletes a user.

Steps to create RESTful Services:

  • i) Create a Spring Boot project
  • ii) Create a REST Controller:
@RestController
@RequestMapping("/a")
public class A
{
    ...
}
  • iii) Run the application
  • iv) Access the API

7) Securing Spring REST Endpoints

  • Securing Spring REST endpoints is an essential part of building secure web applications.
  • It ensures that only authorized users can access the Application and protects the data from attackers.

i) Integration:

  • Spring Security provides built-in mechanisms for authentication and authorization.

    • Authentication:

    The process of verifying the identity of a user or device is called as Authentication.

  • It protects from the common attacks like Injection, session hijacking etc.
  • @EnableWebSecurity annotation is used to provide configuration.

Example:

@EnableWebSecurity
public class A 
{
    @Bean
    ...
}

ii) Custom Security Policies:

  • If built-in policies aren’t enough for your Application, you can create custom security policies to handle your specific needs.
  • @PreAuthorize annotation is used to define access rules.

Example:

@RestController
public class A
{
    @PreAuthorize(...)
    ...
}

iii) Security:

  • Instead of using HTTP, use HTTPS for extra security.
  • JWT or OAuth2 are used for token-based Authentication.
  • JWT stands for JSON Web Token.
